OCR

Optical Character Recognition
Verified: May 20269 sources5 interactive simulations
OCR converts pixels of text in an image — a scan, photo, or PDF — into machine-readable characters a computer can search, copy, and process.

2 Why It Matters

The world is full of text locked inside images: scanned contracts, receipts, license plates, medical records, handwritten forms, old books, road signs. OCR is the bridge that unlocks that text for machines.

Real-World Impact

  • Invoice & Receipt Processing — Accounts-payable automation reads vendor invoices without human data entry.
  • Accessibility — Screen readers use OCR to read text embedded in images for visually impaired users.
  • Search & Archive — Google Books used OCR to digitize 40+ million books. Scanned PDFs become searchable.
  • Banking & KYC — ID card scanning, cheque processing, and form digitization.
  • Autonomous Vehicles — Reading road signs and speed limits in real time.
  • RPA — Bots use OCR to read legacy systems that have no APIs.
  • Translation Apps — Google Lens OCRs text from the camera, then translates it live.
2026 Status OCR is effectively a solved problem for clean typewritten text (>99% accuracy). Research focus has shifted to handwriting, cursive, and complex multi-column layout. Vision-Language Models (VLMs) like Mistral OCR and Qwen2.5-VL are beginning to replace traditional OCR pipelines for high-value document understanding.

3 Core Concepts

Image Preprocessing

Cleaning the raw image: binarization, deskewing, noise removal, contrast enhancement, DPI normalization.

Analogy: Ironing a crumpled page before reading it.

Text Detection

Finding WHERE text exists. Outputs bounding boxes around lines/words. Modern systems use CNN-based detectors (CRAFT, DBNet).

Analogy: Highlighting every sentence before reading.

Segmentation

Breaking detected regions into lines, words, or characters for the recognizer.

Analogy: Cutting a text strip into individual Scrabble tiles.

Feature Extraction (CNN)

A CNN scans the image and extracts visual features: edges, curves, loops, strokes that combine into character shapes.

Analogy: Learning to read by first noticing vertical lines, then curves, then letters.

Sequence Recognition (LSTM + CTC)

BiLSTM reads the feature sequence in both directions. CTC decoder maps the sequence to final characters without per-character alignment.

Analogy: Filling in an ambiguous word by reading surrounding context.

Post-Processing

Dictionary lookups, regex validation, confidence thresholding. Flags low-confidence results for human review.

Analogy: Phone autocorrect — "teh" → "the" from context.

4 How It Works — Step by Step

1

Image Input

Source: scanned document (TIFF/PNG), photo (JPEG), or PDF page rasterized at 300 DPI. Resolution is critical — below 150 DPI accuracy drops sharply. 300 DPI is the minimum standard; 600 DPI for handwriting.

2

Preprocessing

Binarization: Convert to black/white via Otsu's or adaptive thresholding.
Deskewing: Rotate to align text horizontally — skew >5° destroys line detection.
Denoising: Gaussian/median filters remove scanner artifacts.
Normalization: Resize character height to a standard target (e.g., 32px).

3

Text Detection

A detector model (DBNet, CRAFT, EAST) draws bounding boxes around text regions. Modern systems detect at word or line level — separating "where is text?" from "what does it say?". Each box gets a confidence score.

4

Feature Extraction (CNN)

Each text region passes through a CNN backbone (ResNet, VGG). Early layers detect edges and strokes. Deeper layers combine them into character shapes. The output is a feature map — a sequence of vectors the LSTM reads next.

5

Sequence Recognition (BiLSTM + CTC)

The feature sequence feeds into a Bidirectional LSTM. It reads context left-to-right AND right-to-left — knowing surrounding characters helps resolve ambiguous ones (is that an l, 1, or I?). CTC decoder collapses the output into final characters.

6

Post-Processing

Language model scoring, dictionary lookup, regex validation (dates, amounts, IDs), and confidence scoring. Results below threshold are flagged for human review. Domain rules add another validation layer.

7

Output

Plain text, structured JSON (bounding boxes + confidence scores), hOCR format (HTML with position metadata), or searchable PDF. The choice depends on downstream use: RPA needs JSON, search engines need plain text, archival needs searchable PDF.

5 Examples

Example 1 — Tesseract (Simplest, CPU)

# pip install pytesseract pillow opencv-python
# apt-get install tesseract-ocr
import pytesseract
from PIL import Image
import cv2

def preprocess(path):
    img = cv2.imread(path)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    _, binary = cv2.threshold(gray, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
    return Image.fromarray(binary)

text = pytesseract.image_to_string(preprocess("invoice.png"), lang='eng')
print(text)

# With confidence per word:
data = pytesseract.image_to_data(preprocess("invoice.png"), output_type=pytesseract.Output.DICT)
for i, word in enumerate(data['text']):
    if int(data['conf'][i]) > 60:
        print(f"{word:20} {data['conf'][i]}%")

Example 2 — EasyOCR (80+ languages)

# pip install easyocr
import easyocr

reader = easyocr.Reader(['en', 'th'])  # English + Thai
for bbox, text, conf in reader.readtext('document.jpg'):
    print(f"[{conf:.0%}] {text}")

Example 3 — PaddleOCR (best accuracy, complex layouts)

# pip install paddlepaddle paddleocr
from paddleocr import PaddleOCR, PPStructure

ocr = PaddleOCR(use_angle_cls=True, lang='en')
for line in ocr.ocr('invoice.jpg', cls=True)[0]:
    bbox, (text, conf) = line
    print(f"{text:40} ({conf:.1%})")

# Table extraction:
table_engine = PPStructure(table=True, ocr=True, lang='en')
result = table_engine('table.jpg')  # returns HTML table

Example 4 — Accuracy Measurement

from jiwer import cer, wer
ground_truth = "Invoice Total: $1,234.56"
ocr_output   = "lnvoice Total: $1,234.56"
print(f"CER: {cer(ground_truth, ocr_output):.2%}")  # → 4.17%
print(f"WER: {wer(ground_truth, ocr_output):.2%}")  # → 25.00%

6 Common Mistakes & Misconceptions

Mistake 1Skipping preprocessing
Raw photos have glare, distortion, and shadows. Fix: always apply deskew, binarization, denoising. OpenCV's warpPerspective corrects perspective. Adaptive thresholding handles uneven lighting.
Mistake 2Low DPI images (<200 DPI)
At 72 DPI most characters are too pixelated to distinguish. Fix: scan at 300 DPI minimum. If stuck with low-res images, try super-resolution preprocessing (Real-ESRGAN) before OCR.
Mistake 3Trusting OCR output blindly
l/1/I and 0/O are the most common confusions. Also: rn→m, cl→d. Fix: post-process with domain knowledge — amounts fields expect digits only; dates match a regex pattern.
Mistake 4Assuming all OCR handles handwriting
Standard engines are trained on printed text. Handwriting recognition needs ICR/IWR models. Cursive accuracy is still 85–92% vs 99%+ for clean print.
Mistake 5No confidence thresholding
OCR always outputs something even when guessing. A 40% confidence word is essentially random. Fix: threshold at 70–85% and flag anything below for human review.

Key Terms

OCR Quick Reference
CERCharacter Error Rate. Wrong chars ÷ total chars. Primary OCR metric. <1% = excellent.
WERWord Error Rate. Wrong words ÷ total words. Stricter — one bad char breaks the whole word.
DPIDots Per Inch. 300 minimum, 600 for handwriting. 72 DPI (screen) = too low.
BinarizationConvert to pure black/white. Otsu: auto threshold. Adaptive: handles uneven lighting.
DeskewingRotate image to align text horizontally. Skew >5° destroys line detection.
CTCConnectionist Temporal Classification. Trains sequence models without per-character label alignment.
hOCRHTML output format embedding position metadata. Used for searchable PDFs.
DBNet / CRAFTState-of-the-art text detection models. DBNet: fast real-time. CRAFT: curved/artistic text.

8 What to Learn Next

📐

Document Layout Analysis

Detect tables, columns, headers, figures before OCR. Libraries: LayoutParser, PaddleOCR PP-Structure, DocLayNet.

🏷️

Named Entity Recognition

After OCR gives raw text, NER extracts structured info: names, dates, amounts. Combine OCR + NER for invoice automation.

🤖

Vision-Language Models

Qwen2.5-VL, Mistral OCR, InternVL3 read documents as images and output structured data — bypassing traditional OCR pipelines.
